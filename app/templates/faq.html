Frequently Asked Questions

You have two ways to upload data into the database:

  • Via Web Browser: Go to the Upload page, select a .zip archive containing your AF3 result files, and submit. Best for single runs.
  • Via Command Line (CLI): If you have server access, use our powerful CLI tool. You can upload entire directories of outputs simultaneously, assign them to collections, and even use symlinks to save disk space. Example: af3-db upload-folder /path/to/results --collection "My Project".

During the background processing phase, the system identifies chains using one of two strategies:

  • Automatic (Sequence Search): The exact amino acid sequence of each chain is queried against the UniParc/UniProt databases to find matching entries automatically.
  • Filename Pattern: If your output folders follow a strict naming convention (e.g., P12345_Q98765_model), you can provide a Regex-like pattern (like {A}_{B}) during upload. The system will then force-map Chain A to P12345 and Chain B to Q98765.

In addition to parsing the native AlphaFold 3 confidence scores (ipTM, pTM, pLDDT, ranking_score), the backend automatically runs a biophysical analysis on the 3D structure (.cif file). This includes:
  • Buried Surface Area (BSA): The interface area between interacting chains.
  • Hydrogen Bonds & Salt Bridges: Counting specific interactions across the interface.
  • ipSAE & PDockQ: Advanced interface quality and docking metrics derived from the model coordinates.

The Advanced Search allows highly granular filtering:
  • Filter by specific score thresholds using interactive sliders (e.g., only show complexes with ipTM > 0.8).
  • Filter by oligomeric state (Monomer, Homomer, Heteromer).
  • Add chain-specific blocks: You can enforce that a complex must contain a chain matching a specific UniProt ID AND another chain matching a specific gene name.

Collections allow you to group multiple uploads logically (e.g., "Kinase Screen Q3"). Viewing a collection provides aggregated statistics, species distribution charts, scatter plots (like ipTM vs. pTM), and the ability to download the entire dataset as a single bulk ZIP archive.

By default, no. The application is designed to run securely within your local network (LAN) or a virtual private network (VPN). It is only accessible to users who have network access to the host machine. Public internet access depends entirely on how your IT department or the person hosting the database configures the server routing.

If you opted to use local HTTPS during installation, the system uses a self-signed SSL certificate. Browsers display a "Not Secure" or "Your connection is not private" warning because the certificate was generated locally, not by a recognized public authority. This is completely normal and safe for local networks. You can usually bypass this warning by clicking "Advanced" and proceeding to the site.

To remove the application, simply run the uninstall.py script located in your main installation folder. This script will safely stop and remove the Docker containers, unbind the CLI commands, and give you the option to permanently delete the PostgreSQL database and all uploaded structural files.
